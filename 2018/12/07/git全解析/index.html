git全解析

发表于 2018-12-07

常用命令

git init 在当前目录下创建本地仓库
git add . 当前所有修改添加到暂存区
git add 【文件全名称】 指定文件添加到暂存区
git commit -m 'message' 暂存区提交到当前分支
git status 查看当前状态
git diff 查看修改对比，一般借助可视化的工具或编辑器插件

git log 查看仓库的所有提交历史记录
git log --pretty=oneline 查看记录的简略信息
git reset --hard HEAD^ 回退到上一个版本，HEAD指向当前版本，^有几个，就回退到前几个版本。也可用HEAD~xx回退到指定前几个版本
git reset --hard [commit id] 指向已存在的某个版本，回退/往前都可。commit id 可通过log查看。id 填前几位就可
git reflog 查看仓库的所有命令记录，可找到历史的commit id和提交信息

工作区和暂存区

工作区是除了.git隐藏文件夹的所有
.git文件夹就是版本库
版本库最重要的是名为stage(或index)的暂存区
git add把文件修改添加到暂存区
git commit提交更改，实际上就是一次性把暂存区的所有修改的内容提交到当前分支。提交完stage就是干净的
git add 添加的是修改记录，不是当个文件

重要命令

git ckeckout -- [文件全名] 把文件在工作区的修改全部撤销。如果文件的修改还没有被添加到暂存区，就会直接撤销工作区的修改。如果已经提交过到stage，再来修改，则回退到上次add时的状态。没有--就变成了切换分支
git reset HEAD [文件名] 继续上面，如果修改已经被add到暂存区，还没有被commit时。可以把暂存区的修改撤销掉扔回到工作区
当代码已经commit。工作区删除了文件，删除也会视为一种修改
git rm [文件名] 已经add到暂存区的文件的删除其实也是修改操作。可以选择git rm 确认删除，然后就可commit。也可以选择git checkout – [文件名] 回退到上次add的状态

远程仓库

git remote add origin git@github.com:michaelliao/learngit.git 对于已经git init的本地仓库，通过git remote add [远程仓库名] [远程库的http/ssh地址]建立远程连接。git remote remove origin删除连接。使用ssh的方式需要远程账号添加ssh白名单
git push --set-upstream origin master 推送本地到远程。第一次远程库是空的，需要加上-u origin master，这样不仅把本地master分支push，也将本地master和远程master关联了

分支管理

HEAD指向当前分支

每一次的提交组成一条线，master指向最新的提交。再用HEAD指向master，就能确定当前分支以及当前分支的提交点。每次提交，master分支都会向前移动一步

创建新的分支dev时，git创建了一盒新的指针叫dev，指向master相同的提交，再把HEAD指向dev，就表示当前在dev分支上

这样创建一个分支很快，并且工作区没有任何变化
之后的修改都是在dev分支上

每一次的修改dev都往前，master不变
当dev上所有工作已完成，要把dev合并到master上。直接将master指向dev的当前提交就完成了合并

所以，分支的合并很简单，改改指针，工作区不变
合并完后甚至可以删除dev分支

删除分支就是删除dev的指针

基本命令

git branch 查看本地分支
git branch -a 查看本地+远程分支
git branch xx 创建xx分支
git checkout xx 切换到xx分支
git checkout -b xx 创建并切换到xx分支
git merge xx 用于合并指定分支到当前分支
git branch -d xx 删除分支
git branch -D xx 删除一个还没有被合并的分支
git log --graph 查看分支合并图

git stash 把工作区’储蓄’起来，等以后恢复现场时使用
git stash apply 恢复到上次的现场，然后需要删除最后一个stash，使用git stash drop
git stash pop 恢复到最后一次并删除stash
git stash list 查看所有stash
git stash apply stash@{0} 先list查看所有，再恢复到指定stash

多人协作

git remote 查看远程分支名
git remote -v 显示更详细
git checkout -b xx origin/dev 创建并切换到xx分支。同时将xx与dev建立远程连接
git tag <name> 在当前分支的当前commit打版本标签。标签是个快照，不可移动
